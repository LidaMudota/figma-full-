# IC Repair — статический лендинг

Небольшой промо-сайт для студии ремонтных услуг: геро-секция, описание, список услуг, отзывы и форма заявки. Дизайн опирается на тёмную палитру и акцентный золото-керамический цвет.

## Быстрый взгляд
- Верстка без сборки: только `index.html`, `style.css` и папка `images/`.
- Шрифты грузятся с Google Fonts, дополнительного окружения не требуется.
- 3D-ховеры на карточках услуг и метриках, мягко отключаются при `prefers-reduced-motion`.

## Структура проекта
- `index.html` — разметка страниц и мини-скрипт для 3D-наклона карточек.
- `style.css` — базовая типографика, сетки секций, эффекты кнопок и карточек.
- `images/` — локальные изображения для фона и контента.

## Требования
- Современный браузер (Chrome/Firefox/Safari/Edge).
- Для локального запуска нужен только статический сервер (можно обойтись без установки зависимостей).

## Локальный запуск
### A) VS Code Live Server
1. Откройте проект в VS Code.
2. Установите расширение **Live Server** (Ritwick Dey), если оно ещё не установлено.
3. Кликните `Go Live` в статус-баре или `Alt+L Alt+O`.

## Deployment
- **GitHub Pages**: 
  1. Создайте ветку `gh-pages` или включите Pages в настройках репозитория.
  2. Выберите источник `Deploy from a branch` → `gh-pages` или папку `/ (root)` в ветке `main`.
  3. Дождитесь публикации и откройте сгенерированный URL.
- **Любой статический хостинг** (Vercel/Netlify/S3/статический nginx): загрузите `index.html`, `style.css` и папку `images/` в корень хостинга, убедитесь, что путь к файлам не меняется.

## Notes
- 3D-анимации работают на карточках услуг и числовых показателях. Они отключаются для пользователей с включённым `prefers-reduced-motion` и не активируются на устройствах с грубым вводом (touch/pen).
- При отключённом JavaScript остаются CSS-ховеры, базовая верстка не изменяется.
